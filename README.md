# SmartVenue AI Assistant 

SmartVenue AI is a simple yet intelligent assistant designed to improve the experience of people attending large-scale events like sports matches or concerts.

Instead of just giving static answers, it actually tries to **understand the situation and guide users smartly** — helping them avoid crowds, reduce waiting time, and move efficiently inside the venue.

---

## 🚀 The Problem

If you’ve ever been to a big event, you know the struggle:
- Long queues at entry gates  
- Confusion about which route to take  
- Crowded food stalls with long waiting times  

There’s usually **no real-time guidance**, and people end up wasting time.

---

## 💡 The Idea

What if there was a smart assistant that could:
- Tell you which gate is less crowded  
- Suggest faster food options  
- Guide you based on the current situation  

That’s exactly what **SmartVenue AI Assistant** does.

---

## 🧠 What Makes It Special?

This is not just a basic chatbot.

It can:
- Understand follow-up questions like *“now?”* or *“where is it?”*  
- Handle confusing or incomplete queries  
- Suggest better options instead of just answering  
- Deal with edge cases like *“all gates are crowded”*  
- Maintain conversation context like a real assistant  

👉 It behaves more like a **decision-making assistant** than a simple bot.

---

## ⚙️ Tech Stack

- Google Antigravity (prompt-driven AI logic) 
- HTML  
- CSS  
- JavaScript  

---

## 🔍 How It Works

1. You ask a question (e.g., *“Which gate is less crowded?”*)  
2. The system understands what you mean  
3. It checks the available (simulated) crowd data  
4. It gives you the best possible suggestion  

Everything is lightweight and runs smoothly without any backend.

---

## ✨ Features

- 💬 Chat-based interface  
- 🧠 Context-aware responses  
- ⚠️ Crowd alerts and smart suggestions  
- ⏱️ Queue and waiting time guidance  
- 🔄 Follow-up understanding (like “now?”, “what about food?”)  
- 📍 Location-based responses  

---

## 🧪 Example Interactions

- “Which gate is less crowded?” → Suggests best gate  
- “All gates are crowded” → Suggests waiting with prediction  
- “now?” → Uses previous context  
- “where is it?” → Gives correct location  

---

## 🔐 Assumptions

- Crowd and waiting time data are simulated  
- No real-time sensors or APIs are used  
- The goal is to demonstrate logic and decision-making
